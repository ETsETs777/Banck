# Техническое задание: улучшения Spektors Desktop (Launcher)

**Версия документа:** 1.0  
**Объект:** приложение `apps/spektors-desktop` (Electron + React, electron-vite).

## 1. Цель

Сделать десктоп-лаунчер стабильным инструментом ежедневной разработки: предсказуемый запуск, понятные ошибки, безопасность, дистрибуция и сопровождение.

## 2. Текущее состояние (кратко)

- Окно без системной рамки на Windows/Linux, кастомная шапка в renderer.
- IPC: файловая система, профиль, валидация корня репо, запуск команд, Docker-сценарии.
- Сборка: `electron-vite build`; установщик Windows: NSIS (`npm run dist:win`), ярлыки на рабочем столе и в меню «Пуск».

## 3. Функциональные улучшения

1. **Обнаружение окружения**  
   Проверка наличия `docker`, `node`, `npm`/`pnpm` с отображением версий и ссылок на установку при отсутствии.

2. **Журнал операций**  
   Единая панель логов с фильтром по уровню (info/warn/error), экспорт в файл, ограничение объёма (ротация).

3. **Пресеты сценариев**  
   Сохраняемые пользователем наборы шагов («поднять всё», «только API», «миграции») с порядком выполнения.

4. **Health-check сервисов**  
   Агрегированный статус портов 3000–3003 и 8000 с интервалом опроса и ручным обновлением.

5. **Офлайн-режим UI**  
   Явное состояние «сеть недоступна» для ссылок на localhost без ложных таймаутов в UX.

## 4. Нефункциональные требования

| Область | Требование |
|--------|-------------|
| Производительность | Холодный старт окна ≤ 3 с на SSD; без лишних фоновых опросов при свёрнутом окне (опционально). |
| Надёжность | Любой сбой дочернего процесса не должен вешать UI; таймауты на `run-command`. |
| Безопасность | Не логировать секреты из `.env`; ограничить `openExternal` доверенным списком URL (опционально). |
| Локализация | RU/EN для системных сообщений установщика и критичных диалогов. |
| Обновления | (Этап 2) автообновление через `electron-updater` + публикация релизов. |

## 5. Технический долг и рефакторинг

- Вынести повторяющуюся логику путей и IPC в небольшие модули в `src/main/`.
- Типизировать контракты IPC (общие типы main ↔ preload ↔ renderer).
- E2E: smoke-сценарий запуска упакованного `.exe` (Playwright или ручной чеклист в CI).

## 6. Критерии приёмки (MVP улучшений)

- Установка через NSIS создаёт ярлык на рабочем столе и в меню «Пуск» с иконкой приложения.
- После установки приложение запускается, открывается главное окно, сохраняются границы окна.
- Документированы команды: `npm run dev -w spektors-desktop`, `npm run dist:win -w spektors-desktop` (из корня монорепо: `npm run dev:desktop`, `npm run dist:desktop`).

## 7. Развёртывание (Windows)

- **Сборка установщика:** из корня монорепо `npm run dist:desktop` или в приложении `npm run dist:win -w spektors-desktop`.
- **Артефакты:** `apps/spektors-desktop/release/Spektors Launcher-<версия>-Setup.exe`, распакованная копия в `release/win-unpacked/` (для теста без установки).
- **Версия Electron:** в `electron-builder.yml` задано поле `electronVersion` — при обновлении `electron` в репозитории синхронизировать с фактической версией в `node_modules/electron` (в workspaces пакет часто лежит в корне `node_modules`).
- **Локальная сборка без режима разработчика Windows:** для установщика используется `win.signAndEditExecutable: false`, чтобы не тянуть распаковку `winCodeSign` с symlink (иначе возможна ошибка прав доступа). Для публикации пользователям планируется отдельная подпись (см. п. 7 вне скоупа).
