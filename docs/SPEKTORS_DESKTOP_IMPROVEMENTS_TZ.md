# ТЗ: улучшение Spektors Desktop (итерация v0.2)

**Версия документа:** 1.0 · **март 2026**  
**База:** `apps/spektors-desktop`, исходное ТЗ — [DESKTOP_LAUNCHER_TZ.md](./DESKTOP_LAUNCHER_TZ.md).

## 1. Цели

1. **Визуал:** интерфейс должен восприниматься как цельный инструмент разработчика: читаемая типографика, понятная иерархия блоков, аккуратные состояния кнопок и индикаторы доступности сервисов.
2. **Возможности:** сократить рутину — проверка «жив ли порт» перед открытием ссылки, работа с логами (очистка, копирование, автоскролл), открытие корня репозитория в проводнике, запоминание размеров окна.

## 2. Визуальные требования

| Элемент | Требование |
|--------|------------|
| Шрифты | Подключить внешние шрифты (Google Fonts): геометрический sans для UI, моноширинный для путей и логов. Обновить CSP под `fonts.googleapis.com` / `fonts.gstatic.com`. |
| Палитра | Сохранить тёмную тему; уточнить акцент и фон (CSS-переменные), лёгкая сетка/градиент без перегруза. |
| Карточки | Секции с коротким визуальным маркером (иконка или цветная полоса), единые отступы и скругления. |
| Ссылки на сервисы | Индикатор «онлайн / офлайн» (точка или бордер) по результату проверки HTTP с главного процесса. |
| Лог | Фиксированная максимальная высота, прокрутка, явные действия «Очистить» и «Копировать». |

## 3. Функциональные требования (новое / уточнение)

1. **Проверка URL** — IPC `net:probeUrl`: GET-запрос с таймаутом; успех при установлении соединения и ответе (для локальных dev-серверов).
2. **Открыть папку** — IPC `shell:openPath`: открыть выбранный корень репозитория в файловом менеджере ОС.
3. **Состояние окна** — сохранять `getBounds()` в `userData` при закрытии окна; восстанавливать при следующем запуске.
4. **Лог** — автоскролл к последнему сообщению при добавлении записи; кнопки очистки и копирования всего текста в буфер.

## 4. Вне скоупа этой итерации

- Убийство фонового процесса API по PID.
- Встроенный просмотр таблиц БД.
- `electron-builder` и подпись установщика.

## 5. Критерии приёмки

- `npm run dev:desktop` открывает окно с обновлённым UI.
- При запущенном локальном сервисе индикатор у соответствующей ссылки показывает доступность; при остановленном — недоступность.
- «Открыть папку репо» открывает каталог в проводнике / Finder.
- После изменения размера окна и перезапуска размеры восстанавливаются.
- Лог очищается и копируется по кнопкам без ошибок.
